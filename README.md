# Keyword Classifier and Google Ads Campaign Generator

LLM-assisted keyword processing pipeline for Google Ads campaign structuring.

This project takes raw keyword dumps for a product, applies rule-based filtering and LLM classification, and exports a production-ready workbook with:
- campaign/ad-group structure,
- positive keywords,
- negatives with explicit exclusion reasons,
- fingerprint grouping views for auditability.

## What This Project Does

Given a product keyword file (for example, `Myaxyl Oil.xlsx`) and a positioning source-of-truth document, the pipeline:
1. Cleans and normalizes keywords.
2. Applies strict pre-LLM safety/relevance filters.
3. Classifies intent and funnel stage with LLM.
4. Detects language and translates Hindi/Hinglish to English equivalents.
5. Canonicalizes English equivalents semantically with LLM for synonym-aware grouping.
6. Builds semantic fingerprint groups.
7. Assigns match types with broad-match cap control.
8. Applies brand and competitor guardrails.
9. Runs QA + reconciliation and exports final Excel.

## Current Pipeline Notes

- Early informational filtering is not done in Step 2.
- Informational handling is centralized in classification (`Informational` -> negative).
- Phrase-dedup steps are not part of the active pipeline.
- Competitor brands are read from the positioning sheet `Competitor brands` column.
- Competitor non-informational keywords are routed to competitor ad group in Step 8.

## Project Structure

- `main.py`: Primary production pipeline.
- `prepare_input.py`: Converts raw product Excel into `input/input_keywords.csv`.
- `classify.py`: Legacy/auxiliary classifier (not used by `main.py` pipeline).
- `requirements.txt`: Python dependencies.
- `.env`: API key configuration.
- `input/Test_Positioning document.xlsx`: Product context source-of-truth.
- `input/input_keywords.csv`: Prepared input generated by `prepare_input.py`.
- `output/`: Generated output workbooks.

## Requirements

- Python 3.8+
- OpenRouter API key

Install dependencies:

```bash
pip install -r requirements.txt
```

Create `.env` in project root:

```env
OPENROUTER_API_KEY=your_openrouter_api_key_here
```

## Input Contracts

### 1. Raw product keyword file (root folder)
- File name must match product name, for example: `Myaxyl Oil.xlsx`
- Must contain a `keyword` column header.

### 2. Positioning document
- Path: `input/Test_Positioning document.xlsx`
- Required columns:
  - `Product Name`
  - `Website links`
- Optional but strongly used:
  - `Primary use case`
  - `Use case and benefits`
  - `Key benefits`
  - `Competitor brands`

### 3. Prepared CSV
- Generated by `prepare_input.py`
- Required columns for `main.py`:
  - `product_name`
  - `keyword`

## How To Run

From project root:

```bash
python prepare_input.py --product "Myaxyl Oil"
python main.py --product "Myaxyl Oil"
```

Output file will be created in `output/` as:
- `output/<small_product_name>.xlsx` (example: `output/myaxyl.xlsx`)

## Detailed Pipeline Walkthrough (`main.py`)

Execution entrypoint: `process_product(...)`

1. Startup checks
- Builds LLM client using `OPENROUTER_API_KEY`.
- Loads product context from positioning workbook.
- Verifies product URL validity.

2. Step 1 - Keyword cleaning (`step_1_keyword_cleaning`)
- Normalizes case/punctuation/spacing.
- Drops blanks.
- Keeps first normalized duplicate as canonical and moves dropped duplicates to negatives.

3. Step 2 - Pre-LLM filtering (`step_2_filter_keywords`)
Filters:
- marketplace terms,
- non-purchase behavioral queries,
- wrong product format,
- cross-product leakage,
- ambiguous/off-use-case queries.

Note: competitor terms are not pre-negatived here.

4. Step 3 - LLM classification (`step_3_classification`)
LLM returns:
- `intent_type` in `Informational | Commercial | Transactional | Navigational`
- `funnel_stage` in `BOF | MOF | TOF`
- `ad_group_base` in `product_search | brand_search | condition_search | evaluation_search | competitor_search`

Pipeline enforcement:
- informational intent -> negative,
- competitor terms force competitor ad-group base,
- rule guard keeps obvious informational keywords out of active set.

5. Step 3b - Language detection + translation (`translate_keywords_batch`)
- Detects `original_language` as `english | hinglish | hindi`.
- Non-English keywords get `english_equivalent` via LLM.

6. Step 3c - Semantic canonicalization (`canonicalize_english_equivalents_batch`)
- LLM produces `english_equivalent_canonical` for synonym-aware grouping.
- This reduces splits like minor phrasing variants.

7. Step 4 - Fingerprint semantic grouping (`step_1b_fingerprint_semantic_dedup`)
- LLM builds fingerprint objects.
- Core-vs-soft grouping logic applied.
- Fingerprint identity uses canonicalized English meaning to improve semantic grouping.

8. Step 5 - Match type assignment (`step_4_match_types`)
Assigns `Exact | Phrase | Broad` with broad capped at 10%.

9. Step 6 - Filter confirmation (`step_6_filter_confirmation`)
Active keywords must be only:
- `Commercial`, `Transactional`, `Navigational`

10. Step 7 - Brand rules (`step_7_brand_rules`)
- Standalone `kerala ayurveda` is negatived.

11. Step 8 - Competitor rules (`step_8_competitor_rules`)
- Competitor + non-informational intent => positive routed to `competitor_search_<slug>`.
- Competitor + informational intent => negative.

12. Step 9 - Campaign structure (`step_9_build_campaign_structure`)
Creates campaign and ad-group template:
- `google_search_products_<slug>`
- `product_search_<slug>`
- `brand_search_<slug>`
- `condition_search_<slug>`
- `evaluation_search_<slug>`
- `competitor_search_<slug>`

13. Quality checks (`step_10_quality_checks`)
Fails if:
- duplicate keyword across ad groups,
- broad share > 10%,
- informational term leaks into active list.

14. Reconciliation
- Enforces `input count == positives + negatives`.
- Prevents silent keyword loss.

15. Export (`export_output`)
Writes:
- `CAMPAIGN_STRUCTURE`
- `KEYWORDS`
- `FINGERPRINT_DEDUP`
- `FINGERPRINT_SUMMARY`

## Output Sheets

### `CAMPAIGN_STRUCTURE`
Campaign and ad-group blueprint with intent and description.

### `KEYWORDS`
Operational table with keyword, polarity, match type, intent, funnel stage, ad group, notes, fingerprint, and language fields.

### `FINGERPRINT_DEDUP`
Audit fingerprint view with `Fingerprint Group ID`; includes dedup-style notes for repeated fingerprint members.

### `FINGERPRINT_SUMMARY`
Grouped fingerprint analytics with group id, fingerprint JSON, keyword count, and keyword list.

## Troubleshooting

1. `PermissionError` on output `.xlsx`
- Close the workbook in Excel (and preview pane), then rerun.
- Excel creates lock files like `~$glymin.xlsx`.

2. Git add fails on `output/~$*.xlsx`
- Lock/temp files are now ignored by `.gitignore`.

3. `Connection error` / `getaddrinfo failed`
- Network/DNS issue while calling OpenRouter.
- Retry after network stabilizes.

4. Reconciliation failure
- Pipeline intentionally stops if counts mismatch.
- Inspect negative reasons and step decisions.

5. Product not found
- Ensure product name in command matches positioning sheet.
